### Whiteboard naming

When navigating this app to add or access information, it is unlikely I will want to navigate to some Category or Subject (though I do want to be able to associate a card with one, so it can be referenced and searched for). Instead, I will have in my mind the *why* behind the information. For this reason, I should not use whiteboards primarily to bucket notes by their category (or type, etc); instead, I should use whiteboard that are titled in a very expressive way, having parent whiteboards that add context to that expression. Exception to this is the Library, where I will want to look up by subject.
